## 动态规划

我先定义 dp 状态。
然后分析当前位置的答案由哪些更小子问题转移而来，写出状态转移方程。
接着确定初始条件和遍历顺序。
最后看是否可以做空间优化。

### 动态规划通常用在这类问题里：

问题可以拆成子问题
子问题会重复出现
每个子问题的最优解，可以推出原问题的最优解

1. 定义状态

先想清楚：

dp[i] 或 dp[i][j] 到底表示什么？

这是最核心的一步。

比如爬楼梯：

- dp[i] = 爬到第 i 阶楼梯的方法数

比如最长公共子序列：

- dp[i][j] = text1[:i] 和 text2[:j] 的最长公共子序列长度

2. 找状态转移方程

也就是：

当前状态怎么由以前状态推出来？

比如爬楼梯：

- dp[i] = dp[i-1] + dp[i-2]

因为到第 i 阶，只可能：

- 从 i-1 走一步上来
- 从 i-2 走两步上来

3. 初始化

转移之前，最开始那些状态得先知道。

比如爬楼梯：
```
dp[1] = 1
dp[2] = 2
```
4. 遍历顺序

你推 dp[i] 的时候，依赖的状态必须已经算出来。

这就是为什么很多题遍历顺序很关键。

### 背包问题

- 背包问题的核心是定义状态 dp[j] 或 dp[i][j]，表示在一定物品范围内、容量限制下的最优值。
01 背包中每个物品只能选一次，所以一维优化时必须倒序遍历容量，避免同一轮重复使用当前物品。
完全背包中每个物品可以无限次选，所以要正序遍历容量，让当前物品的贡献可以继续传递。
多重背包则是每个物品最多选若干次，朴素做法是枚举取几个，进一步可以通过二进制拆分把它转成若干个 01 物品，从而优化复杂度。

- 同一个转移公式，区别只是容量循环方向，本质由“当前物品能不能重复使用”决定。

不能重复用：倒序，01 背包
可以重复用：正序，完全背包
有次数上限：枚举次数，或拆成 01 背包
